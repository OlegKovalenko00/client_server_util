# Тестовое задание

Требуется разработать ряд консольных приложений:

- клиент для отправки файлов на проверку;
- сервер для проверки файлов на наличие вредоносных паттернов;
- утилиту для отображения статистики обнаруженных угроз.

## Клиент

При запуске программе в аргументах командной строки передаётся путь до файла, который необходимо проверить, и номер порта.

Программа должна:

- подключиться к серверу по указанному порту;
- передать выбранный файл на проверку в текстовом виде;
- после получения результата проверки вывести его на консоль и завершить выполнение.

## Сервер

При запуске программе в аргументах командной строки передаётся путь до конфигурационного файла и номер порта.

В конфигурационном файле содержится список вредоносных паттернов. Формат конфигурации можно выбрать любой: `CSV`, `JSON`, `ini` и т.д.

Программа должна:

- считать конфигурационный файл и запустить прослушивающий сервер на указанном порту;
- для каждого подключающегося клиента создавать отдельный процесс, который будет взаимодействовать с этим клиентом;
- проверить полученный файл на наличие вредоносных паттернов из конфигурационного файла;
- сообщить клиенту результат проверки и закрыть соединение;
- вести статистику обнаружения угроз и по запросу передавать её утилите для отображения.

Допускается сперва сохранить файл для последующего анализа, усложнённая версия подразумевает потоковую обработку.

Статистика должна включать:

- количество проверенных файлов;
- обнаруженные вредоносные паттерны и их количество.

При получении сигнала завершения программа должна:

- закрыть все соединения;
- дождаться завершения всех процессов, взаимодействующих с клиентами;
- завершить выполнение.

## Утилита для отображения статистики

При запуске программа выполняет подключение к серверу по IPC-каналу и запрашивает текущую статистику.

IPC-взаимодействие необходимо реализовать с использованием именованных каналов:

- Windows: `Named Pipe`
- Linux: `FIFO`

После получения статистики программа выводит её на консоль и завершает выполнение.

## Условия

- приложения должны быть написаны с использованием `C++17` и выше;
- приложения должны быть написаны для `Linux` или `Windows` на выбор;
- для работы с сетью и IPC разрешается использование только системных библиотек, сторонние библиотеки, такие как `Boost.Asio`, запрещены;
- для работы с форматами конфигурационного файла и статистики разрешается использование сторонних библиотек.

## Плюсом будут

- использование системы сборки при разработке: `CMake`, `Bazel` и др.;
- наличие unit-тестов для каждого приложения. Для написания тестов может быть использован `GTest` или любой другой фреймворк.
