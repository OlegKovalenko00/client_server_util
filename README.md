##Задание:
#Требуется разработать ряд консольных приложений:
· Клиент для отправки файлов на проверку.
· Сервер для проверки файлов на наличие вредоносных паттернов.
· Утилита для отображения статистики обнаруженных угроз.
#Клиент
При запуске программе в аргументах командной строки передается путь до файла, который
необходимо проверить, и номер порта.
Программа должна подключиться к серверу по указанному порту и передать выбранный файл на
проверку в текстовом виде.
После получения результата проверки от сервера программа должна вывести его на консоль и
завершить выполнение.
#Сервер
При запуске программе в аргументах командной строки передается путь до конфигурационного
файла и номер порта.
В конфигурационном файле содержится список вредоносных паттернов, формат можно выбрать
любой (CSV, JSON, ini и т.д.).
Программа должна считать конфигурационный файл и запустить прослушивающий сервер на
указанном порту.
Для каждого подключающегося клиента программа должна создавать отдельный процесс, который
будет взаимодействовать
с этим клиентом. Программа должна проверить полученный файл на наличие вредоносных
паттернов из конфигурационного
файла, после чего сообщить клиенту результат проверки и закрыть соединение. Допускается сперва
сохранить файл для
последующего анализа, усложненная версия подразумевает потоковую обработку.
Программа также должна вести статистику обнаружения угроз и по запросу передавать ее утилите
для отображения.
Формат для передачи стаистики можно выбрать любой. Статистика включает в себя следующие
значения:
· Количество проверенных файлов.
· Обнаруженные вредоносные паттерны и их количество.
При получении сигнала завершения программа должна закрыть все соединения, дождаться
завершения всех процессов,
взаимодействующих с клиентами, после чего завершить выполнение.
#Утилита для отображения статистики
При запуске программа выполняет подключение к серверу по IPC каналу и запрашивает текущую
статистику.
IPC взамодействие необходимо реализовать с использованием именнованных каналов (Windows -
Named Pipe, Linux - FIFO).
После получения статистики от сервера программа выводит ее на консоль и завершает выполнение.
##Условия:
· Приложения должны быть написаны с использованием С++17 и выше.
· Приложения должны быть написаны для ОС Linux или Windows на выбор.
· Для работы с сетью и IPC разрешается использование только системных библиотек,
сторонние библиотеки, такие как Boost.Asio и др. запрещены.
· Для работы с форматами конфигурационного файла и статистики разрешается
использование сторонних библиотек
